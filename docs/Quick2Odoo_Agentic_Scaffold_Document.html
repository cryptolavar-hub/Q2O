Project Overview

The Quick2Odoo Agentic Scaffold is a sophisticated multi-agent development system designed to automate the creation of a complete SaaS application for migrating data from QuickBooks to Odoo v18. This system leverages specialized AI agents that work collaboratively to generate, test, and quality-assure production-ready code across multiple technology stacks.

The system is built to handle complex enterprise requirements including multi-tenant SaaS architecture, OAuth integrations, real-time data synchronization, workflow orchestration, infrastructure as code, and comprehensive security implementations.

What Makes This System Unique

  • Specialized Agents: Each agent type is optimized for specific domains (infrastructure, integration, frontend, workflow, security)
  • Template-Based Generation: Uses Jinja2 templates for maintainable, customizable code generation
  • Configurable Project Layouts: Flexible directory structures adaptable to different project needs
  • Comprehensive Quality Assurance: Automated testing, security scanning, and code quality validation
  • Production-Ready Output: Generates deployable code with proper error handling, logging, and documentation

Key Features

1. Multi-Agent Architecture

Specialized Agent Types

The system includes 9 specialized agent types, each optimized for specific tasks:

  • Orchestrator Agent: Manages task breakdown and distribution
  • Coder Agent: Generates FastAPI endpoints and SQLAlchemy models
  • Testing Agent: Creates and executes pytest test suites
  • QA Agent: Performs code quality reviews
  • Infrastructure Agent: Generates Terraform and Helm configurations
  • Integration Agent: Creates OAuth and API client code
  • Frontend Agent: Generates Next.js/React components
  • Workflow Agent: Creates Temporal workflow definitions
  • Security Agent: Performs security audits

2. Intelligent Task Management

  • Automatic Task Breakdown: Orchestrator analyzes objectives and creates manageable tasks
  • Dependency Resolution: Automatic dependency detection and ordering
  • Progress Tracking: Real-time status updates and completion percentages
  • Error Handling: Graceful failure handling with detailed error reports

3. Template-Based Code Generation

  • Jinja2 Templates: Maintainable, reusable template system
  • Fallback Support: Inline templates as fallback if external templates unavailable
  • Customizable: Easy to modify templates for project-specific needs
  • Type-Safe: Generates properly typed code with Pydantic models

4. Quality Assurance

  • Automated Testing: pytest-based test generation and execution
  • Static Analysis: Integration with ruff, mypy, bandit, and safety
  • Code Quality Metrics: Complexity analysis, style checking, and best practice validation
  • Infrastructure Validation: Terraform and Helm configuration validation

5. Configurable Project Layouts

  • Flexible Structure: Configurable directory layouts via ProjectLayout class
  • Default Presets: Sensible defaults for standard projects
  • JSON Configuration: Load custom layouts from configuration files
  • Multi-Project Support: Different layouts for different project types

6. CI/CD Integration

  • GitHub Actions: Pre-configured CI/CD pipeline
  • Multi-Version Testing: Tests across Python 3.10, 3.11, 3.12
  • Coverage Reports: Codecov integration for coverage tracking
  • Security Scanning: Automated security checks on every commit

System Architecture

Agent System Overview

The system follows a coordinator-worker pattern where the Orchestrator Agent breaks down high-level objectives into tasks and distributes them to specialized agents.

Orchestrator Agent

Role: Project coordinator and task manager

Responsibilities:

  • Analyzes project objectives and context
  • Detects objective types (infrastructure, integration, frontend, workflow, API)
  • Identifies technology stacks from objectives
  • Creates tasks with appropriate agent assignments
  • Manages task dependencies
  • Tracks project progress and completion status

Coder Agent

Role: Backend code generation

Responsibilities:

  • Generates FastAPI endpoints with Pydantic models
  • Creates SQLAlchemy data models with multi-tenant support
  • Implements service layer business logic
  • Generates API documentation
  • Uses Jinja2 templates for code generation

Infrastructure Agent

Role: Infrastructure as Code generation

Responsibilities:

  • Generates Terraform configurations for Azure
  • Creates Helm charts for Kubernetes deployments
  • Configures WAF policies and security rules
  • Sets up Application Insights and monitoring
  • Validates generated configurations

Integration Agent

Role: External API integration code

Responsibilities:

  • Generates QuickBooks OAuth 2.0 flows
  • Creates Odoo XML-RPC client code
  • Implements Stripe billing integration
  • Generates webhook handlers
  • Creates API client wrappers

Frontend Agent

Role: Frontend UI generation

Responsibilities:

  • Generates Next.js pages and routes
  • Creates React components with TypeScript
  • Implements authentication flows
  • Generates theme toggle components
  • Creates onboarding wizards

Workflow Agent

Role: Temporal workflow generation

Responsibilities:

  • Creates Temporal workflow definitions
  • Generates activity implementations
  • Creates worker configurations
  • Implements backfill and sync workflows

Testing Agent

Role: Test generation and execution

Responsibilities:

  • Generates pytest test files
  • Executes tests using pytest
  • Collects test results and coverage
  • Generates test reports

QA Agent

Role: Code quality assurance

Responsibilities:

  • Reviews code for quality issues
  • Checks code style and formatting
  • Validates error handling
  • Provides improvement recommendations

Security Agent

Role: Security auditing

Responsibilities:

  • Scans code for security vulnerabilities
  • Checks for hardcoded secrets
  • Validates authentication implementations
  • Reviews OAuth flows for security issues

Data Flow

  1. Project Input: User provides project description and objectives
  2. Task Breakdown: Orchestrator analyzes and creates tasks
  3. Task Distribution: Tasks assigned to appropriate agents
  4. Code Generation: Agents generate code using templates
  5. Validation: Testing and QA agents validate generated code
  6. Output: Generated files written to workspace directory

Installation & Setup

Prerequisites

  • Python: 3.10, 3.11, or 3.12
  • pip: Latest version
  • Git: For version control (optional but recommended)
  • Terraform: 1.6.0+ (optional, for infrastructure validation)
  • Helm: 3.13.0+ (optional, for Helm validation)

Installation Steps

1. Clone or Download the Project

git clone <repository-url>
cd QuickOdoo

2. Create Virtual Environment

# On Windows
python -m venv venv
venv\Scripts\activate

# On Linux/Mac
python3 -m venv venv
source venv/bin/activate

3. Install Dependencies

pip install --upgrade pip
pip install -r requirements.txt
Note: The installation includes all necessary packages with pinned versions for reproducibility.

4. Verify Installation

python -c "from agents import OrchestratorAgent; print('Installation successful!')"

Optional: Install Infrastructure Tools

For infrastructure validation, install Terraform and Helm:

Terraform

  • Windows: Download from terraform.io/downloads
  • Linux: sudo apt-get install terraform
  • Mac: brew install terraform

Helm

  • Windows: choco install kubernetes-helm
  • Linux: Follow Helm installation guide
  • Mac: brew install helm

Usage Guide

Command Line Usage

Basic Usage

# Run with command-line arguments
python main.py --project "QuickBooks to Odoo Integration" \
               --objective "User authentication" \
               --objective "Data synchronization"

Using Configuration File

Create a JSON configuration file:

{
  "project_description": "QuickBooks to Odoo Online Integration SaaS",
  "objectives": [
    "OAuth authentication with QuickBooks",
    "Odoo v18 integration",
    "Stripe billing setup",
    "Terraform infrastructure",
    "Next.js frontend"
  ]
}

Then run:

python main.py --config config.json --workspace ./my_project

Command Line Options

Option Description Example
--project Project description --project "My Project"
--objective Project objective (can be repeated) --objective "Feature 1"
--config Path to JSON config file --config config.json
--workspace Workspace directory path --workspace ./output
--log-level Logging level (DEBUG, INFO, WARNING, ERROR) --log-level DEBUG
--output Path to save results JSON --output results.json

Programmatic Usage

from main import AgentSystem

# Initialize the system
system = AgentSystem(workspace_path="./my_workspace")

# Run a project
results = system.run_project(
    project_description="QuickBooks to Odoo Integration",
    objectives=[
        "OAuth authentication",
        "Data synchronization",
        "Frontend dashboard"
    ]
)

# Access results
print(f"Completion: {results['final_status']['completion_percentage']}%")
print(f"Tasks completed: {results['final_status']['completed']}")
print(f"Tasks failed: {results['final_status']['failed']}")

Troubleshooting

Common Issues and Solutions

Issue: Import Errors

Symptoms: ModuleNotFoundError or ImportError

Solutions:

  • Verify virtual environment is activated: which python (should show venv path)
  • Reinstall dependencies: pip install -r requirements.txt --force-reinstall
  • Check Python version: python --version (should be 3.10+)
  • Verify PYTHONPATH includes project root

Issue: Template Not Found

Symptoms: TemplateNotFound error

Solutions:

  • Check that templates/ directory exists in project root
  • Verify template files have .j2 extension
  • System will fall back to inline templates if external templates unavailable
  • Check file permissions on template files

Issue: Tests Fail to Execute

Symptoms: pytest errors or test execution failures

Solutions:

  • Install pytest: pip install pytest pytest-asyncio pytest-cov
  • Check that generated code is importable: python -c "from api.app import endpoints"
  • Verify PYTHONPATH: export PYTHONPATH=$PWD:$PYTHONPATH
  • Run tests manually: pytest tests/ -v

Issue: Terraform/Helm Validation Fails

Symptoms: Infrastructure validation errors

Solutions:

  • Verify Terraform is installed: terraform version
  • Verify Helm is installed: helm version
  • Validation is optional - system will skip if tools not available
  • Check generated Terraform files for syntax errors
  • Run Terraform validation manually: terraform fmt -check

Issue: Windows Path Issues

Symptoms: Path separator errors on Windows

Solutions:

  • Use os.path.join() instead of hardcoded paths (already implemented)
  • Check that paths use forward slashes or raw strings: r"C:\path\to\file"
  • System uses pathlib.Path where possible for cross-platform compatibility

Issue: Permission Denied

Symptoms: File write permission errors

Solutions:

  • Check workspace directory permissions
  • Run with appropriate user permissions
  • Verify disk space available
  • Check antivirus isn't blocking file writes

Issue: Agent Not Processing Tasks

Symptoms: Tasks remain in PENDING status

Solutions:

  • Check task dependencies - tasks may be blocked
  • Review logs for errors: --log-level DEBUG
  • Verify agent is registered with orchestrator
  • Check that task type matches agent capabilities
  • Increase max iterations if needed (default: 100)

Glossary

Technical terms and concepts used in this documentation:

Term Definition
Agent A specialized software component that performs specific tasks in the multi-agent system. Each agent has a specific role (e.g., code generation, testing, infrastructure).
FastAPI A modern Python web framework for building APIs. Used in this system for generating REST API endpoints.
Jinja2 A templating engine for Python. Used to generate code from templates, making code generation maintainable and customizable.
Orchestrator The central coordinator agent that breaks down projects into tasks and distributes them to specialized agents.
Terraform An Infrastructure as Code (IaC) tool used to define and provision cloud infrastructure using declarative configuration files.
Helm A package manager for Kubernetes that simplifies deployment of applications using charts (pre-configured Kubernetes resources).
OAuth 2.0 An authorization framework that enables applications to obtain limited access to user accounts on HTTP services. Used for QuickBooks integration.
Pydantic A Python library for data validation using Python type annotations. Used in FastAPI for request/response validation.
SQLAlchemy A Python SQL toolkit and Object-Relational Mapping (ORM) library. Used for database interactions and model definitions.
Temporal A workflow orchestration platform that ensures reliable execution of distributed applications. Used for data synchronization workflows.
Next.js A React framework for building web applications. Used for generating the frontend user interface.
pytest A Python testing framework that makes it easy to write simple and scalable test cases.
SSE (Server-Sent Events) A web standard that enables servers to push data to web browsers. Used for real-time job status updates.
WAF (Web Application Firewall) A security solution that monitors and filters HTTP traffic to and from web applications. Configured via Terraform in this system.
Multi-Tenant An architecture where a single instance of software serves multiple customers (tenants), with data isolation between tenants.
XML-RPC A remote procedure call (RPC) protocol using XML. Used by Odoo for API communication.
Workspace The directory where generated code and project files are created.
Task A unit of work assigned to an agent. Tasks have status (pending, in_progress, completed, failed, blocked) and dependencies.
Template A Jinja2 template file (.j2) that defines the structure and content of generated code.
Project Layout A configuration that defines the directory structure for generated files (e.g., where API code, frontend files, infrastructure configs are placed).
CI/CD Continuous Integration/Continuous Deployment - automated processes for testing and deploying code changes.
Static Analysis Analysis of code without executing it, used to find bugs, security issues, and code quality problems.
IaC (Infrastructure as Code) The practice of managing and provisioning infrastructure through machine-readable definition files, rather than manual configuration.
Webhook An HTTP callback that allows external systems to notify your application of events (e.g., Stripe payment events).
