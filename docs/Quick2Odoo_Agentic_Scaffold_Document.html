System Architecture

Agent System Overview

The system follows a coordinator-worker pattern where the Orchestrator Agent breaks down high-level objectives into tasks and distributes them to specialized agents. The system includes multiple instances of each agent type for high availability, with a load balancer managing task routing and failover.

High Availability: Each agent type has primary + backup instances. The load balancer automatically routes tasks to healthy agents and fails over when needed.

System Components

  1. AgentSystem: Main coordinator that initializes all agents and manages the project lifecycle
  2. OrchestratorAgent: Breaks down projects into tasks and manages task distribution
  3. Specialized Agents: 10 agent types handling specific domains
  4. LoadBalancer: Routes tasks to agents with health checks and failover
  5. Message Broker: Enables inter-agent communication
  6. Dashboard: Real-time monitoring and metrics
  7. Utilities: Template rendering, language detection, security scanning, etc.

Data Flow

  1. Project Input: User provides project description and objectives
  2. Task Breakdown: Orchestrator analyzes and creates tasks with dependencies
  3. Task Distribution: Load balancer routes tasks to appropriate agents
  4. Code Generation: Agents generate code using templates
  5. Quality Checks: Testing and QA agents validate generated code
  6. VCS Integration: Auto-commit and PR creation (if enabled)
  7. Dashboard Updates: Real-time progress broadcast to dashboard
  8. Output: Generated files written to workspace directory

1. PlatformMapper (utils/platform_mapper.py) v3.0

Purpose

Universal data transformer that converts ANY accounting platform's data format to Odoo format using JSON mapping configurations.

Features

  • JSON-Based Mapping: Loads field mappings from config/*_to_odoo_mapping.json
  • Nested Field Handling: Handles complex nested fields (e.g., BillAddr.Line1)
  • Transformation Types: Composite fields, lookups, date conversions, boolean inversions
  • Entity Relationship Preservation: Maintains foreign key mappings (QB ID ‚Üí Odoo ID)
  • Platform-Specific Transformations: Specialized methods for each entity type
  • Multi-Platform Support: QuickBooks, SAGE, Wave, and extensible

Key Methods

  • transform_entity(entity_type, data) - Generic transformation
  • transform_customer(qb_customer) - QB Customer ‚Üí Odoo res.partner
  • transform_invoice(qb_invoice) - QB Invoice ‚Üí Odoo account.move with lines
  • transform_account(qb_account) - QB Account ‚Üí Odoo account.account with type mapping

2. MigrationOrchestrator (utils/migration_orchestrator.py) v3.0

Purpose

Coordinates complete end-to-end migration from any platform to Odoo with error handling and validation.

Features

  • 4-Phase Migration: Extract ‚Üí Transform ‚Üí Load ‚Üí Validate
  • Entity Mapping Cache: Maintains Source ID ‚Üí Odoo ID relationships
  • Migration Sequencing: Ensures master data migrates before transactions
  • Error Handling: Per-entity error catching without stopping entire migration
  • Validation: Entity count matching, balance verification
  • Comprehensive Reporting: JSON migration report with statistics and errors

Migration Workflow

  1. Extract: Call source_client.get_all_entities()
  2. Transform: Use PlatformMapper to convert to Odoo format
  3. Load: Create records in Odoo maintaining relationships
  4. Validate: Verify counts, balances, trial balance
  5. Report: Generate detailed migration report

Quality Over Speed

  • Isolated Development: Each platform has unique quirks - handle them independently
  • Thorough Testing: Platform-specific sandbox testing ensures quality
  • Risk Mitigation: Bugs are isolated to single platform
  • Incremental Value: Deliver working integrations progressively
  • Learning & Improvement: Each integration improves the adapter pattern
  • Code Reuse: ~60% code reuse across platforms

Troubleshooting

Common Issues and Solutions

Issue: Import Errors

Symptoms: ModuleNotFoundError or ImportError

Solutions:

  • Verify virtual environment is activated: which python (should show venv path)
  • Reinstall dependencies: pip install -r requirements.txt --force-reinstall
  • Check Python version: python --version (should be 3.10+)
  • Verify PYTHONPATH includes project root

Issue: Template Not Found

Symptoms: TemplateNotFound error

Solutions:

  • Check that templates/ directory exists in project root
  • Verify template files have .j2 extension
  • System will fall back to inline templates if external templates unavailable
  • Check file permissions on template files

Issue: Tests Fail to Execute

Symptoms: pytest errors or test execution failures

Solutions:

  • Install pytest: pip install pytest pytest-asyncio pytest-cov
  • Check that generated code is importable: python -c "from api.app import endpoints"
  • Verify PYTHONPATH: export PYTHONPATH=$PWD:$PYTHONPATH
  • Run tests manually: pytest tests/ -v

Issue: Terraform/Helm Validation Fails

Symptoms: Infrastructure validation errors

Solutions:

  • Verify Terraform is installed: terraform version
  • Verify Helm is installed: helm version
  • Validation is optional - system will skip if tools not available
  • Check generated Terraform files for syntax errors
  • Run Terraform validation manually: terraform fmt -check

Issue: Windows Path Issues

Symptoms: Path separator errors on Windows

Solutions:

  • Use os.path.join() instead of hardcoded paths (already implemented)
  • Check that paths use forward slashes or raw strings: r"C:\path\to\file"
  • System uses pathlib.Path where possible for cross-platform compatibility

Issue: Permission Denied

Symptoms: File write permission errors

Solutions:

  • Check workspace directory permissions
  • Run with appropriate user permissions
  • Verify disk space available
  • Check antivirus isn't blocking file writes

Issue: Agent Not Processing Tasks

Symptoms: Tasks remain in PENDING status

Solutions:

  • Check task dependencies - tasks may be blocked
  • Review logs for errors: --log-level DEBUG
  • Verify agent is registered with orchestrator
  • Check that task type matches agent capabilities
  • Increase max iterations if needed (default: 100)
  • Check load balancer status - agent may be unhealthy

Issue: Dashboard Not Connecting

Symptoms: Dashboard WebSocket connection fails

Solutions:

  • Verify dashboard server is running: python -m api.dashboard.main
  • Check firewall settings for port 8001
  • Verify WebSocket URL: ws://localhost:8001/ws/dashboard
  • Check browser console for errors

Issue: VCS Integration Not Working

Symptoms: No auto-commits or PR creation

Solutions:

  • Verify VCS_ENABLED=true environment variable is set
  • Check Git is available: git --version
  • Verify workspace is a Git repository or will be initialized
  • For PR creation, verify GITHUB_TOKEN and GITHUB_REPO are set
  • Check GitHub token has appropriate permissions

Issue: Load Balancer Not Routing Tasks

Symptoms: Tasks not being assigned to agents

Solutions:

  • Check agent health status via load balancer
  • Verify agents are registered with load balancer
  • Check agent capacity limits - may be at capacity
  • Review load balancer logs for routing decisions
  • Verify circuit breakers are not open for all agents

Debug Mode

python main.py --log-level DEBUG --project "Test" --objective "Feature"

Verification Steps

  1. Check Installation: 
    python -c "from agents import *; from utils import *; print('OK')"
  2. Test Template System: 
    python -c "from utils import get_renderer; r = get_renderer(); print(r.template_exists('api/fastapi_endpoint.j2'))"
  3. Test Project Layout:

Glossary

Technical terms and concepts used in this documentation:

Term Definition
Agent A specialized software component that performs specific tasks in the multi-agent system. Each agent has a specific role (e.g., code generation, testing, infrastructure).
Agent Redundancy Multiple instances of the same agent type for high availability and failover capabilities.
Bandit A Python security linter that scans code for security vulnerabilities and common issues.
Circuit Breaker A design pattern that prevents cascading failures by stopping requests to a failing service after a threshold is reached.
FastAPI A modern Python web framework for building APIs. Used in this system for generating REST API endpoints.
Helm A package manager for Kubernetes that simplifies deployment of applications using charts (pre-configured Kubernetes resources).
Jinja2 A templating engine for Python. Used to generate code from templates, making code generation maintainable and customizable.
Load Balancer A component that distributes tasks across multiple agent instances, ensuring optimal resource utilization and high availability.
Message Broker A middleware component that enables communication between agents using pub/sub messaging patterns.
MyPy A static type checker for Python that helps catch type-related errors before runtime.
Multi-Tenant An architecture where a single instance of software serves multiple customers (tenants), with data isolation between tenants.
Node.js A JavaScript runtime built on Chrome's V8 engine. The system supports Node.js 20.x LTS for server-side JavaScript development.
OAuth 2.0 An authorization framework that enables applications to obtain limited access to user accounts on HTTP services. Used for QuickBooks integration.
Orchestrator The central coordinator agent that breaks down projects into tasks and distributes them to specialized agents.
Pydantic A Python library for data validation using Python type annotations. Used in FastAPI for request/response validation.
Pub/Sub Publish/Subscribe messaging pattern where senders (publishers) send messages without knowing specific recipients (subscribers).
pytest A Python testing framework that makes it easy to write simple and scalable test cases.
Retry Policy A configuration that defines how tasks should be retried on failure, including retry count, delays, and exception handling.
Ruff A fast Python linter and code formatter that checks code style and identifies potential issues.
Semgrep A static analysis tool that uses pattern matching to find security vulnerabilities and code issues.
SQLAlchemy A Python SQL toolkit and Object-Relational Mapping (ORM) library. Used for database interactions and model definitions.
SSE (Server-Sent Events) A web standard that enables servers to push data to web browsers. Used for real-time job status updates.
Temporal A workflow orchestration platform that ensures reliable execution of distributed applications. Used for data synchronization workflows.
Terraform An Infrastructure as Code (IaC) tool used to define and provision cloud infrastructure using declarative configuration files.
Next.js A React framework for building web applications. Used for generating the frontend user interface.
Task A unit of work assigned to an agent. Tasks have status (pending, in_progress, completed, failed, blocked) and dependencies.
Template A Jinja2 template file (.j2) that defines the structure and content of generated code.
Project Layout A configuration that defines the directory structure for generated files (e.g., where API code, frontend files, infrastructure configs are placed).
VCS (Version Control System) Version control system (typically Git) for tracking changes and managing code versions. The system supports automatic commits and PR creation.
WAF (Web Application Firewall) A security solution that monitors and filters HTTP traffic to and from web applications. Configured via Terraform in this system.
Webhook An HTTP callback that allows external systems to notify your application of events (e.g., Stripe payment events).
WebSocket A communication protocol that provides full-duplex communication over a single TCP connection. Used for real-time dashboard updates.
Workspace The directory where generated code and project files are created.
XML-RPC A remote procedure call (RPC) protocol using XML. Used by Odoo for API communication.
CI/CD Continuous Integration/Continuous Deployment - automated processes for testing and deploying code changes.
Static Analysis Analysis of code without executing it, used to find bugs, security issues, and code quality problems.
IaC (Infrastructure as Code) The practice of managing and provisioning infrastructure through machine-readable definition files, rather than manual configuration.
ESM (ECMAScript Modules) The standard module system for JavaScript. Node.js 20.x fully supports ESM modules.
LTS (Long Term Support) A software release that receives maintenance and security updates for an extended period. Node.js 20.x is an LTS release.
